# web

Frontend for accept-payments — a Vite + React + TypeScript SPA using
[TanStack Router](https://tanstack.com/router) (file-based) and
[shadcn/ui](https://ui.shadcn.com) on [Base UI](https://base-ui.com).

## Develop

```bash
bun install
bun dev            # http://localhost:5173
```

The receipt route is `/success?session_id=<id>` — Stripe Checkout's
`success_url` redirects here after payment.

### Preview without a backend (dev only)

```
/success?demo=paid     # settled card receipt
/success?demo=unpaid   # ACH "processing" receipt
```

### Real data

The SPA fetches `GET /sessions/:id`; in dev Vite proxies `/sessions` → the Rust
Lambda, so there's no CORS to configure. Run a local backend in another terminal:

```bash
cargo lambda watch --env-file .env.local   # repo root — serves the Lambda on :9000
```

…or point at a deployed URL:

```bash
API_PROXY_TARGET=https://<lambda-url> bun dev
```

The Lambda's `GET /sessions/:id` returns
`{ id, payment_status, amount_total, currency }`.

## Build

```bash
bun run build      # tsc + vite build
```

`src/routeTree.gen.ts` is generated by the TanStack Router Vite plugin (on
`dev`/`build`) and committed so a clean checkout type-checks without a prior run.

## Production

CI builds the SPA and bakes `dist/` into the Lambda binary
(`cargo lambda build --features embed-web`); the Lambda serves it from its HTTPS
Function URL, same-origin with the API — no separate static host or CORS.
